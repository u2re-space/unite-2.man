# Recovery Checklist

Status legend:
- [ ] Not started
- [~] In progress
- [x] Done
- [!] Blocked

## Checkpoints

- [ ] Inventory surviving files, UI, docs, configs, scripts, and assets.
- [ ] Discover and document build/run/test commands.
- [ ] Make the project build successfully.
- [ ] Make the app launch with existing UI/UX preserved.
- [ ] Connect existing UI connect/start action to CWSP service/core.
- [ ] Display connection/session status in the existing UI.
- [ ] Implement or verify one local/LAN clipboard sync path.
- [ ] Add or verify basic logs/errors for manual debugging.
- [ ] Document current state, known issues, and next steps.
